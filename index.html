Como remover o aviso de segurança da Conexão de Área de Trabalho Remota (RDP) no Windows

Guia rápido para parar o pop-up "Cuidado: conexão remota desconhecida / Fornecedor desconhecido" que aparece a cada conexão.

Antes de começar: esses passos são feitos na máquina Windows que você usa para CONECTAR (o cliente), não no computador remoto. Você precisa estar logado como Administrador.

Existem dois avisos diferentes — descubra o seu

  • Caso A — O aviso mostra "Fornecedor desconhecido" e uma lista de recursos (cartão, área de transferência, impressoras). → Você abre um arquivo .rdp salvo. Use a Solução A.
  • Caso B — O aviso fala que "a identidade do computador remoto não pode ser verificada" (certificado). → Você digita o IP/nome direto no programa. Use a Solução B.

Na dúvida, faça os dois — não há conflito.

Solução A — aviso "Fornecedor desconhecido" (arquivo .rdp)

Isso libera o Windows a abrir arquivos .rdp sem assinatura, sem o aviso.

Opção 1 — PowerShell (recomendado)

  1. Menu Iniciar → digite PowerShell.
  2. Clique com o botão direito → Executar como administrador.
  3. Cole o bloco abaixo e aperte Enter:
$p = "HKLM:\SOFTWARE\Policies\Microsoft\Windows NT\Terminal Services"
New-Item -Path $p -Force | Out-Null
Set-ItemProperty -Path $p -Name "AllowUnsignedFiles" -Type DWord -Value 1
Set-ItemProperty -Path $p -Name "AllowSignedFiles"   -Type DWord -Value 1

Opção 2 — Prompt de Comando (CMD)

  1. Menu Iniciar → digite cmd → Executar como administrador.
  2. Cole as duas linhas abaixo e aperte Enter:
reg add "HKLM\SOFTWARE\Policies\Microsoft\Windows NT\Terminal Services" /v AllowUnsignedFiles /t REG_DWORD /d 1 /f
reg add "HKLM\SOFTWARE\Policies\Microsoft\Windows NT\Terminal Services" /v AllowSignedFiles   /t REG_DWORD /d 1 /f
✅ Pronto. Não precisa reiniciar. Abra o .rdp de novo — o aviso não deve mais aparecer.

Conferir se aplicou

Rode (não precisa ser admin só para ler):

reg query "HKLM\SOFTWARE\Policies\Microsoft\Windows NT\Terminal Services" /v AllowUnsignedFiles

Tem que retornar AllowUnsignedFiles   REG_DWORD   0x1.

Solução B — aviso de certificado (conectando por IP/nome)

Jeito fácil (1 clique)

Quando o aviso de certificado aparecer, marque a caixinha "Não perguntar novamente para conexões com este computador" e clique em Sim/Conectar. Some para aquele computador.

Jeito definitivo (todos os computadores)

Desliga a verificação que gera o aviso de certificado. PowerShell como administrador:

$c = "HKCU:\Software\Microsoft\Terminal Server Client"
New-Item -Path $c -Force | Out-Null
Set-ItemProperty -Path $c -Name "AuthenticationLevelOverride" -Type DWord -Value 0

Equivalente em CMD:

reg add "HKCU\Software\Microsoft\Terminal Server Client" /v AuthenticationLevelOverride /t REG_DWORD /d 0 /f
⚠️ Sobre segurança: esses ajustes desligam um aviso que existe para proteger contra conexões falsas. Faça isso apenas quando você confia no computador remoto (ex.: sua própria máquina, em rede segura/VPN como Tailscale). Em rede pública ou máquina de terceiros, prefira o "jeito fácil" (caixinha por computador).

⭐ Jeito recomendado — certificar de verdade (em vez de desligar o aviso)

Para o aviso de certificado (Caso B), o melhor não é desligar a verificação, e sim fazer o seu computador confiar de verdade na máquina remota. O aviso some porque a identidade passou a ser legítima — mais seguro, não menos. São 3 passos.

✅ Recomendado quando você vai usar sempre a mesma máquina remota (ex.: seu host via Tailscale). Substitui o "jeito definitivo" do Caso B.

Passo 1 — na máquina REMOTA: exportar o certificado

PowerShell como administrador no computador remoto. Exporta só a parte pública (seguro de compartilhar):

$c = Get-ChildItem Cert:\LocalMachine\"Remote Desktop"
Export-Certificate -Cert $c[0] -FilePath C:\rdp-host.cer

Passo 2 — copiar o arquivo para a máquina do cliente

Leve o C:\rdp-host.cer até a máquina que conecta (Gustavo) — pen drive, Tailscale, e-mail, o que for. Ele não é segredo.

Passo 3 — na máquina do CLIENTE: instalar como confiável

PowerShell como administrador na máquina do Gustavo (ajuste o caminho se salvou em outra pasta):

Import-Certificate -FilePath C:\rdp-host.cer -CertStoreLocation Cert:\LocalMachine\Root
✅ Pronto. Na próxima conexão o aviso de certificado não aparece mais — porque a máquina remota agora é confiável de verdade. Não precisa do AuthenticationLevelOverride.
Para desfazer este passo: remova o certificado importado na máquina do cliente — abra certlm.msc → Autoridades de Certificação Raiz Confiáveis → Certificados → localize o certificado do host e exclua.

Como desfazer (voltar os avisos)

reg delete "HKLM\SOFTWARE\Policies\Microsoft\Windows NT\Terminal Services" /v AllowUnsignedFiles /f
reg delete "HKLM\SOFTWARE\Policies\Microsoft\Windows NT\Terminal Services" /v AllowSignedFiles /f
reg delete "HKCU\Software\Microsoft\Terminal Server Client" /v AuthenticationLevelOverride /f

Ainda aparece algum aviso?

  • Confirme que abriu o PowerShell/CMD como administrador (sem isso, a escrita falha).
  • Veja exatamente qual aviso aparece e identifique se é Caso A ou Caso B acima.
  • Feche e abra o programa de Conexão Remota de novo após aplicar.
